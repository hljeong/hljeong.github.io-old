a - wizard of orz

problem statement

there are \(n\) panels placed in a straight line. each panel can display digits 0 through 9, initially displaying 0.

each second, the digit displayed on each panel is increased by 1 (digit 9 becomes digit 0).

one can pause exactly one panel at a chosen time. when a panel is paused, its digit will stop increasing.

every second after that, unpaused panels adjacent to paused panels become paused. in other words, the pausing propagates towards the left and the right at 1 panel per second.

find the maximum possible number the digits on the panels represent after all panels have paused.

solution

note that we always can and want to make the first panel display 9.

suppose our strategy is to pause the first panel when it is displaying 9. this would be optimal for the case where \(n = 1\), but when \(n \geq 2\), we will obtain some number of the form \(90\dots\), which is worse than pausing the second panel when it is displaying 8, which gives us \(98\dots\).

suppose \(n \geq 3\) and we pause some panel after the second panel while making sure that the first panel displays 9. then we will obtain some number of the form \(987\dots\), which is, again, worse than pausing the second panel when it is displaying 8, which gives us \(989\dots\).

therefore we pause the first panel at 9 when \(n = 1\) and pause the second panel at 8 when \(n \geq 2\).

code

c++ \(\mathcal O(n)\) 
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(0);
    cin.tie(0);

    int t;
    cin >> t;
    while (t --> 0) {
        int n;
        cin >> n;

        // we always want the first digit to be 9
        cout << '9';

        for (int i = 1; i < n; ++i) {
            // from the second digit on, display 89012...
            cout << (8 + (i - 1)) % 10;
        }
        cout << endl;
    }
}

b - hills and valleys

problem statement

given array \(a\) of length \(n\), define a hill to be

  • an index \(i\) such that \(2 \leq i \leq n\), \(a_i > a_{i - 1}\), and \(a_i > a_{i + 1}\)

and a valley to be

  • an index \(i\) such that \(2 \leq i \leq n\), \(a_i < a_{i - 1}\), and \(a_i < a_{i + 1}\).

find the minimum total number of hills and valleys in \(a\) after changing at most one value.

solution

note that changing a value to one of its adjacent values is always at least as good as changing it to some other value.

also note that changing a single value affects at most three hills and/or valleys, therefore we can simply try changing every value to its (up to 2) adjacent values in \(\mathcal O(n)\).

code

c++ \(\mathcal O(n)\) 
#include <bits/stdc++.h>
using namespace std;

const int MAXN = 3e5 + 5;

int n, a[MAXN];

bool hill_or_valley(int i) {
    return a[i] > a[i - 1] && a[i] > a[i + 1] ||
           a[i] < a[i - 1] && a[i] < a[i + 1];
}

// count the number of hills and valleys in i - 1, i, and i + 1
int count(int i) {
    int ans = 0;
    if (i >= 2) {
        ans += hill_or_valley(i - 1);
    }
    if (i >= 1 && i < n - 1) {
        ans += hill_or_valley(i);
    }
    if (i < n - 2) {
        ans += hill_or_valley(i + 1);
    }
    return ans;
}

void minimize(int &a, const int b) {
    if (b < a) {
        a = b;
    }
}

int main() {
    ios::sync_with_stdio(0);
    cin.tie(0);

    int t;
    cin >> t;
    while (t --> 0) {
        cin >> n;
        for (int i = 0; i < n; ++i) {
            cin >> a[i];
        }

        // find the number of hills and valleys
        int original_total = 0, ans;
        for (int i = 1; i < n - 1; ++i) {
            original_total += hill_or_valley(i);
        }
        ans = original_total;

        for (int i = 0; i < n; ++i) {
            int original_value = a[i], original_count = count(i);

            if (i >= 1) {
                a[i] = a[i - 1];
                minimize(ans, original_total - original_count + count(i));
            }

            if (i < n - 1) {
                a[i] = a[i + 1];
                minimize(ans, original_total - original_count + count(i));
            }

            a[i] = original_value;
        }

        cout << ans << endl;
    }
}

c - three bags

problem statement

given 3 non-empty multisets of numbers, one can, in one operation,

  • select \(a\) from one multiset and \(b\) from a different multiset,
  • remove \(b\) from the second multiset, and
  • replace \(a\) in the first multiset with \(a - b\).

find the maximum possible number such that it is the only number left after a sequence of the described operation.

solution

we can think of a sequence of the operation as a tree.

each element in the original multisets correspond to a node in the tree, the root of the tree is the last element to be replaced, and the children of each node are the elements that are subtracted from it. note that the children of each node must be from a different multiset.

the final number is then the sum of elements with even depth minus the sum of elements with odd depth. this will be the value of a tree.

then we want to minimize the sum of elements with odd depth. suppose we already decided the root of the tree. then depth 1 should consist of elements from the two multisets that do not contain the root. note that we can fit all elements in depth 2 if depth 1 has:

  • the minimum element from each of the two multisets, or
  • all elements from one of the two multisets.

the better of the two would yield a maximal value of the tree given a root.

note that no matter which element we choose within the same multiset to be the root, the maximal value of the tree will be the same. then we only need to check 3 possibilities.

code

c++ \(\mathcal O(n_1 + n_2 + n_3)\) 
#include <bits/stdc++.h>
using namespace std;

const int MAXN = 3e5 + 5, INF = 1e9 + 7;

int n[3], a[3][MAXN], minimum[3] = {INF, INF, INF};
long long sum[3];

void minimize(int &a, const int b) {
    if (b < a) {
        a = b;
    }
}

void maximize(long long &a, const long long b) {
    if (b > a) {
        a = b;
    }
}

int main() {
    ios::sync_with_stdio(0);
    cin.tie(0);

    cin >> n[0] >> n[1] >> n[2];
    for (int i = 0; i < 3; ++i) {
        for (int j = 0; j < n[i]; ++j) {
            cin >> a[i][j];
            minimize(minimum[i], a[i][j]);
            sum[i] += a[i][j];
        }
    }

    long long ans = 0;
    for (int i = 0; i < 3; ++i) {
        maximize(ans, sum[i] + sum[(i + 1) % 3] - sum[(i + 2) % 3]);
        maximize(ans, sum[i] + sum[(i + 2) % 3] - sum[(i + 1) % 3]);
        maximize(ans, sum[i] + sum[(i + 1) % 3] + sum[(i + 2) % 3] - 2 * minimum[(i + 1) % 3] - 2 * minimum[(i + 2) % 3]);
    }
    cout << ans << endl;
}

d - sum of paths

problem statement

given a number \(k\) and \(n\) cells arranged in a row, each containing a value \(a_i\), define a good path to be a sequence of cells \(c_0, c_1, \dots\) such that \(c_i\) and \(c_{i + 1}\) are adjacent for all \(i\).

define the value of a good path \(c_0, c_1, \dots\) to be \(a_{c_0} + a_{c_1} + \cdots\).

given \(q\) queries of the form \(i, x\), update \(a_i\) to be \(x\) and output the sum of values of all good paths of length \(k\).

solution

we can calculate for each \(i\) the number of times \(cnt_i\) cell \(i\) appears in a good path of length \(k\). note that this number does not change between queries.

to calculate \(cnt_i\), we first calculate $dp_{i, j} = $ the number of good paths of length \(i\) ending at cell \(j\). note that \(dp_{0, i} = 1\) for all \(i\). the transition is as follows: \[ dp_{i, j} = \begin{cases} dp_{i - 1, 1} & j = 0 \\ dp_{i - 1, n - 2} & j = n - 1 \\ dp_{i - 1, j - 1} + dp_{i - 1, j + 1}. \end{cases} \]

it takes \(\mathcal O(nk)\) time to compute all of the \(dp\) values.

then \(cnt_i = \sum_{j = 0}^k dp_{j, i} \times dp_{k - j, i}\) is the number of times cell \(i\) appears in a good path of length \(k\). this step also takes \(\mathcal O(nk)\) time.

then for each query, simply update \(a_i\) and the sum in \(\mathcal O(1)\).

code

c++ \(\mathcal O(nk + q)\) 
#include <bits/stdc++.h>
using namespace std;

const int MAXN = 5005, MAXK = 5005, MOD = 1e9 + 7;

int n, k, q, ans = 0, a[MAXN], dp[MAXK][MAXN], cnt[MAXN];

int main() {
    ios::sync_with_stdio(0);
    cin.tie(0);

    cin >> n >> k >> q;
    for (int i = 0; i < n; ++i) {
        cin >> a[i];
        dp[0][i] = 1;
    }

    for (int i = 1; i <= k; ++i) {
        dp[i][0] = dp[i - 1][1];
        dp[i][n - 1] = dp[i - 1][n - 2];
        for (int j = 1; j < n - 1; ++j) {
            dp[i][j] = dp[i - 1][j - 1] + dp[i - 1][j + 1];
            if (dp[i][j] >= MOD) {
                dp[i][j] -= MOD;
            }
        }
    }
    for (int i = 0; i <= k; ++i) {
        for (int j = 0; j < n; ++j) {
            cnt[j] += (int) (((long long) dp[i][j] * dp[k - i][j]) % MOD);
            if (cnt[j] >= MOD) {
                cnt[j] -= MOD;
            }
        }
    }

    for (int i = 0; i < n; ++i) {
        ans += (int) (((long long) a[i] * cnt[i]) % MOD);
        if (ans >= MOD) {
            ans -= MOD;
        }
    }

    while (q --> 0) {
        int i, x;
        cin >> i >> x;
        --i;

        ans += MOD - (int) (((long long) a[i] * cnt[i]) % MOD);
        if (ans >= MOD) {
            ans -= MOD;
        }

        a[i] = x;

        ans += (int) (((long long) a[i] * cnt[i]) % MOD);
        if (ans >= MOD) {
            ans -= MOD;
        }

        cout << ans << endl;
    }
}
