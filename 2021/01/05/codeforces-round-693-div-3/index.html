a - cards for friends

problem statement

given a \(w \times h\) sheet of paper, one can:

  • cut and get two sheets of size \(\frac{w}{2} \times h\) if \(w\) is even.
  • cut and get two sheets of size \(w \times \frac{h}{2}\) if \(h\) is even.

determine whether it is possible to cut the \(w \times h\) sheet of paper into \(\geq n\) pieces.

solution

we can cut lengthwise as long as the width is even and vice versa, and every cut doubles the amount of pieces.

code

c++ \(\mathcal O(\log w + \log h)\) 
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(0);
    cin.tie(0);

    int t;
    cin >> t;
    while (t --> 0) {
        int w, h, n, p = 0;
        cin >> w >> h >> n;

        while (w % 2 == 0) {
            ++p;
            w /= 2;
        }

        while (h % 2 == 0) {
            ++p;
            h /= 2;
        }

        // p is actually never > 31 given that w, h <= 1e4
        if (p > 31 or n <= (1 << p)) {
            cout << "YES" << endl;
        } else {
            cout << "NO" << endl;
        }
    }
}

b - fair division

problem statement

determine whether it is possible to split \(n\) candies each weighing either 1 gram or 2 grams fairly.

solution

let \(cnt1\) be the number of candies of weight 1 and \(cnt2\) be the number of candies of weight 2.

if \(cnt1\) is odd, then it is impossible to split the candies.

otherwise, if \(cnt2\) is even, then both can get \(\frac{cnt1}{2}\) candies of weight 1 and \(\frac{cnt2}{2}\) candies of weight 2.

otherwise, it is possible to split the candies fairly only if \(cnt1 \geq 2\): one gets \(\frac{cnt1 - 2}{2}\) candies of weight 1 and \(\left \lceil \frac{cnt2}{2} \right \rceil\) candies of weight 2, and the other gets \(\frac{cnt1 + 2}{2}\) candies of weight 1 and \(\left \lfloor \frac{cnt2}{2} \right \rfloor\) candies of weight 2.

code

c++ \(\mathcal O(n)\) 
#include <bits/stdc++.h>
using namespace std;

const int MAXN = 105;
int a[MAXN];

int main() {
    ios::sync_with_stdio(0);
    cin.tie(0);

    int t;
    cin >> t;
    while (t --> 0) {
        int n, cnt1 = 0, cnt2 = 0;
        cin >> n;
        for (int i = 0; i < n; ++i) {
            cin >> a[i];

            if (a[i] == 1) {
                ++cnt1;
            } else {
                ++cnt2;
            }
        }

        if (cnt1 % 2 == 1) {
            cout << "NO" << endl;
        } else if (cnt2 % 2 == 0) {
            cout << "YES" << endl;
        } else if (cnt1 >= 2) {
            cout << "YES" << endl;
        } else {
            cout << "NO" << endl;
        }
    }
}

c - long jumps

problem statement

given an array \(a\) of length \(n\):

  • choose a starting index \(i\) (\(1 \leq i \leq n\)).
  • while \(i \leq n\), add \(a_i\) to score and jump \(a_i\) to the right (i.e. \(i = i + a_i\)).
  • the game ends when \(i > n\).

find the maximum possible score.

solution

note that the score you gain after jumping from \(i\) to \(i + a_i\) is the same as the score for starting index \(i + a_i\). since we only jump to the right at each step, we can compute the score for each starting index from right to left with the formula \[ score[i] = \begin{cases} a[i] + score[i + a[i]] & \text{ if } i + a[i] \leq n \\ a[i] & \text{ otherwise}. \end{cases} \]

code

c++ \(\mathcal O(n)\) 
#include <bits/stdc++.h>
using namespace std;

const int MAXN = 2e5 + 5;
int a[MAXN];
long long score[MAXN];

int main() {
    ios::sync_with_stdio(0);
    cin.tie(0);

    int t;
    cin >> t;
    while (t --> 0) {
        int n;
        cin >> n;
        long long ans = 0;
        for (int i = 0; i < n; ++i) {
            cin >> a[i];
        }

        for (int i = n - 1; i >= 0; --i) {
            score[i] = a[i];

            if (i + a[i] < n) {
                score[i] += score[i + a[i]];
            }

            if (score[i] > ans) {
                ans = score[i];
            }
        }
        cout << ans << endl;
    }
}

d - even-odd game

problem statement

given an array \(a\) of length \(n\):

  • players take turns, alice moves first
  • each turn a player removes an element from \(a\)
  • if alice removes an even element, it is added to her score
  • if bob removes an odd element, it is added to his score

find out who will win if both players play optimally.

solution

we claim that the optimal strategy for both players is to remove the largest element each turn.

suppose we are alice, then we win if the difference between our final score and bob's final score is positive.

  • if we remove an even element \(x\), then we add \(x\) to the difference. if bob removes \(x\), then the difference stays the same.
  • if we remove an odd element \(y\), then the difference stays the same. if bob removes \(y\), then bob subtracts \(y\) from the difference.

therefore removing any element \(z\) is \(z\) points more profitable than bob removing \(z\). it follows that we should always the largest such \(z\).

the same argument applies for bob's strategy.

for implementation, simply sort \(a\) in descending order and simulate the game.

code

c++ \(\mathcal O(n \log n)\) 
#include <bits/stdc++.h>
using namespace std;

const int MAXN = 2e5 + 5;
int a[MAXN];
long long score[2];

int main() {
    ios::sync_with_stdio(0);
    cin.tie(0);

    int t;
    cin >> t;
    while (t --> 0) {
        int n;
        cin >> n;
        for (int i = 0; i < n; ++i) {
            cin >> a[i];
        }
        sort(a, a + n, greater<int>());

        score[0] = score[1] = 0;
        for (int i = 0; i < n; ++i) {
            if (a[i] % 2 == i % 2) {
                score[i % 2] += a[i];
            }
        }

        if (score[0] == score[1]) {
            cout << "Tie" << endl;
        } else if (score[0] > score[1]) {
            cout << "Alice" << endl;
        } else {
            cout << "Bob" << endl;
        }
    }
}

e - correct placement

problem statement

there are \(n\) friends, each characterized by two values \(h_i\) and \(w_i\). friend \(j\) can be placed in front of friend \(i\) if any of the following is satisfied:

  • \(h_j < h_i\) and \(w_j < w_i\)
  • \(w_j < h_i\) and \(h_j < w_i\)

for each friend \(i\), find any friend \(j\) that can be placed in front of friend \(i\), if there is any.

solution

let us simplify the problem by duplicating each friend and swapping their dimension, that is, for each friend \(i\), create a new friend \(j\) such that \(h_j = w_i\) and \(w_j = h_i\)

the problem then becomes: for each of the original friend \(i\), find any friend \(j\) such that \(h_j < h_i\) and \(w_j < w_i\). then we can simply find \(j\) such that \(h_j < h_i\) and \(w_j\) is minimum, and compare \(w_j\) with \(w_i\). we can do so using a prefix minimum array or a fenwick tree. note that we need to store both the value \(w_j\) and the index \(j\).

implementation note: since \(h_i\) and \(w_i\) can be as large as \(10^9\), we need to perform coordinate compression (you can see how other people write it here).

code

c++ (prefix minimum array) \(\mathcal O(n \log n)\) 
#include <bits/stdc++.h>
using namespace std;

const int MAXN = 2e5 + 5, INF = 1e9 + 7;
int w[MAXN], h[MAXN], tmp[2 * MAXN];
int pre_val[2 * MAXN], pre_ind[2 * MAXN];

int main() {
    ios::sync_with_stdio(0);
    cin.tie(0);

    int t;
    cin >> t;
    while (t --> 0) {
        int n;
        cin >> n;
        for (int i = 0; i < n; ++i) {
            cin >> w[i] >> h[i];
            tmp[i] = w[i];
            tmp[n + i] = h[i];
        }

        // coordinate compression
        sort(tmp, tmp + 2 * n);
        unordered_map<int, int> compress;
        int id = 0;
        compress[tmp[0]] = id++;
        for (int i = 1; i < 2 * n; ++i) {
            if (tmp[i] != tmp[i - 1]) {
                compress[tmp[i]] = id++;
            }
        }

        // initialize prefix minimum array
        for (int i = 0; i <= id; ++i) {
            pre_val[i] = INF;
            pre_ind[i] = -1;
        }

        for (int i = 0; i < n; ++i) {
            if (h[i] < pre_val[compress[w[i]] + 1]) {
                pre_val[compress[w[i]] + 1] = h[i];
                pre_ind[compress[w[i]] + 1] = i + 1;
            }
            if (w[i] < pre_val[compress[h[i]] + 1]) {
                pre_val[compress[h[i]] + 1] = w[i];
                pre_ind[compress[h[i]] + 1] = i + 1;
            }
        }

        // propagate prefix minimum
        for (int i = 1; i <= id; ++i) {
            if (pre_val[i - 1] < pre_val[i]) {
                pre_val[i] = pre_val[i - 1];
                pre_ind[i] = pre_ind[i - 1];
            }
        }

        for (int i = 0; i < n; ++i) {
            // val is minimum h[j] such that w[j] < w[i]
            int val = pre_val[compress[w[i]]];
            int ind = pre_ind[compress[w[i]]];
            if (val < h[i]) {
                cout << ind << ' ';
            } else {
                cout << -1 << ' ';
            }
        }
        cout << endl;
    }
}
c++ (fenwick tree) \(\mathcal O(n \log n)\) 
#include <bits/stdc++.h>
using namespace std;

const int MAXN = 2e5 + 5, INF = 1e9 + 7;
int m, w[MAXN], h[MAXN], tmp[2 * MAXN];
int fw_val[2 * MAXN], fw_ind[2 * MAXN];

void upd(int i, int val, int ind) {
    ++i;
    while (i < m) {
        if (val < fw_val[i]) {
            fw_val[i] = val;
            fw_ind[i] = ind;
        }
        i += i & -i;
    }
}

void qry(int i, int &val, int &ind) {
    val = INF;
    ind = -1;
    while (i != 0) {
        if (fw_val[i] < val) {
            val = fw_val[i];
            ind = fw_ind[i];
        }
        i -= i & -i;
    }
}

int main() {
    ios::sync_with_stdio(0);
    cin.tie(0);

    int t;
    cin >> t;
    while (t --> 0) {
        int n;
        cin >> n;
        for (int i = 0; i < n; ++i) {
            cin >> w[i] >> h[i];
            tmp[i] = w[i];
            tmp[n + i] = h[i];
        }

        // coordinate compression
        sort(tmp, tmp + 2 * n);
        unordered_map<int, int> compress;
        int id = 0;
        compress[tmp[0]] = id++;
        for (int i = 1; i < 2 * n; ++i) {
            if (tmp[i] != tmp[i - 1]) {
                compress[tmp[i]] = id++;
            }
        }

        // initialize fenwick tree
        for (int i = 0; i <= id; ++i) {
            fw_val[i] = INF;
            fw_ind[i] = -1;
        }

        m = id;
        for (int i = 0; i < n; ++i) {
            upd(compress[w[i]], h[i], i + 1);
            upd(compress[h[i]], w[i], i + 1);
        }

        for (int i = 0; i < n; ++i) {
            // val is minimum h[j] such that w[j] < w[i]
            int val, ind;
            qry(compress[w[i]], val, ind);
            if (val < h[i]) {
                cout << ind << ' ';
            } else {
                cout << -1 << ' ';
            }
        }
        cout << endl;
    }
}

f - new year's puzzle

problem statement

given a \(2 \times n\) strip with some of the cells blocked, check if it is possible to tile the strip with dominoes.

solution

let's simulate tiling the strip from left to right. let \(last\) be the leftmost incompletely tiled column so far, \(profile\) describe the current configuration of dominoes shown below, and \(blockage\) describe the configuration of blocked cells also shown below.

\(profile = 0\)

\(profile = 1\)

\(profile = 2\)

\(blockage = 1\)

\(blockage = 2\)

\(blockage = 3\)

initially, \(last = 0\) and \(profile = 0\). we iterate over columns containing blocked cells from left to right. let \(col\) be the next column that contains blocked cells.

if \(profile = 0\), then it does not change since we can tile dominoes vertically.

if \(profile \neq 0\), then we stagger the dominoes and the new \(profile\) depends on the parity of \((col - last)\).

after updating \(profile\), if \(profile\ \&\ blockage\) is nonzero, \(\&\) being the bitwise AND operator, the dominoes conflict with the blocked cells and we cannot tile the strip.

otherwise, \(profile\) becomes \((3 - profile - blockage)\) and \(last\) becomes \((col + 1)\).

if \(profile \neq 0\) when we reach the end of the strip, then we cannot tile the strip.

this is not the cleanest solution.

code

c++ \(\mathcal O(m \log m)\) 
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(0);
    cin.tie(0);

    int t;
    cin >> t;
    while (t --> 0) {
        int n, m;
        cin >> n >> m;
        vector<pair<int, int>> block;
        for (int i = 0; i < m; ++i) {
            int row, col;
            cin >> row >> col;
            block.push_back(make_pair(col - 1, row - 1));
        }

        sort(block.begin(), block.end());
        int profile = 0, ind = 0, last = 0;
        bool yes = true;
        while (ind < m) {
            if (profile != 0) {
                profile = 1 + (profile - 1 + block[ind].first - last) % 2;
            }
            int blockage = 1 << block[ind].second;
            if (ind + 1 < m && block[ind].first == block[ind + 1].first) {
                blockage = 3;
                ++ind;
            }
            if ((profile & blockage) != 0) {
                yes = false;
                break;
            }
            profile = 3 - (profile + blockage);
            last = block[ind].first + 1;
            ++ind;
        }
        yes &= profile == 0;
        if (yes) {
            cout << "YES" << endl;
        } else {
            cout << "NO" << endl;
        }
    }
}

g - moving to the capital

problem statement

given a directed graph with \(n\) nodes, let \(d_i\) be the shortest distance from node 1 to node \(i\). if one is currently at node \(i\), the person chooses one of the following actions:

  1. move along edge \((i, j)\) if it exists and \(d_i < d_j\).
  2. move along edge \((i, j)\) if it exists and \(d_i \geq d_j\).
  3. stop.

the person cannot choose action 2 more than once.

for each node \(i\) find the minimal value of \(d_j\) where one can get from node \(i\) to node \(j\).

solution

let's call the direction strictly away from node 1, i.e. from node \(i\) to node \(j\) where \(d_j > d_i\), forward, and the direction towards node \(1\), i.e. from node \(i\) to node \(j\) where \(d_j \leq d_i\), backward.

we want to find, for each node \(i\), the minimal value of \(d_j\) where one can get from node \(i\) to node \(j\) by making at most 1 backward move. let's denote this value by \(ans_i\).

we will find \(ans_i\) in descending order of \(d_i\). for each node \(i\) in descending order of \(d_i\) and each neighbor \(j\) of node \(i\):

  • if \(d_j \leq d_i\), i.e. edge \((i, j)\) is backward, minimize \(ans_i\) with \(d_j\).
  • if \(d_j > d_i\), i.e. edge \((i, j)\) is forward, minimize \(ans_i\) with \(ans_j\).

note that we never end up with a path which contains more than one backward edge.

this solution is strikingly similar to my solution for problem c and is essentially dynamic programming.

code

c++ \(\mathcal O(n \log n + m)\) 
#include <bits/stdc++.h>
using namespace std;

const int MAXN = 2e5 + 5;
// queue for bfs
int q[MAXN], q_head, q_tail;
int d[MAXN], ans[MAXN];
bool visited[MAXN];

int main() {
    ios::sync_with_stdio(0);
    cin.tie(0);

    int t;
    cin >> t;
    while (t --> 0) {
        int n, m;
        cin >> n >> m;
        for (int i = 0; i < n; ++i) {
            visited[i] = false;
        }
        vector<vector<int>> graph(n);
        for (int i = 0; i < m; ++i) {
            int u, v;
            cin >> u >> v;
            graph[u - 1].push_back(v - 1);
        }

        // bfs
        q_head = q_tail = 0;
        visited[0] = true;
        d[0] = 0;
        q[q_tail++] = 0;
        while (q_head != q_tail) {
            int node = q[q_head++];
            for (int neighbor : graph[node]) {
                if (!visited[neighbor]) {
                    visited[neighbor] = true;
                    d[neighbor] = d[node] + 1;
                    q[q_tail++] = neighbor;
                }
            }
        }

        vector<pair<int, int>> order;
        for (int i = 0; i < n; ++i) {
            ans[i] = d[i];
            order.push_back(make_pair(d[i], i));
        }

        // traverse in descending order of d[i]
        sort(order.begin(), order.end(), greater<pair<int, int>>());
        for (pair<int, int> entry : order) {
            int node = entry.second;
            for (int neighbor : graph[node]) {
                if (d[neighbor] <= d[node]) {
                    if (d[neighbor] < ans[node]) {
                        ans[node] = d[neighbor];
                    }
                } else {
                    if (ans[neighbor] < ans[node]) {
                        ans[node] = ans[neighbor];
                    }
                }
            }
        }

        for (int i = 0; i < n; ++i) {
            cout << ans[i] << " \n"[i == n - 1];
        }
    }
}
